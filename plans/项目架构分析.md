# 追忆 Reminiscence - 项目架构分析

## 一、项目概述

**追忆**是一个 AI 驱动的数字人永生系统。通过分析聊天记录、音频等数据，学习一个人的说话风格、音色、性格特征，让心中那个人在数字世界中"赛博永生"。

> 此情可待成追忆，只是当时已惘然。

---

## 二、技术栈总览

| 层级 | 技术 | 版本 |
|------|------|------|
| 桌面框架 | Tauri | 2.x |
| 前端 | React + TypeScript + Vite | React 19 / Vite 8 |
| 后端 | Python FastAPI | 0.115.0 |
| 数据库 | SQLite（内置） | - |
| 向量存储 | 自研 SimpleVectorStore（numpy） | - |
| NLP | jieba + SnowNLP | - |
| LLM调用 | OpenAI SDK（兼容接口） | 1.50.0 |
| TTS | MiMo API | - |

## 五、核心模块详解

### 5.1 数据模型层 (`models/data_models.py`)

定义了整个系统的统一数据结构：

| 数据类 | 用途 |
|--------|------|
| `Message` | 统一消息格式，支持多平台来源 |
| `ChatSession` | 聊天会话 |
| `LanguageStyle` | 语言风格特征（句长、口头禅、表情偏好、标点习惯等） |
| `PersonalityTraits` | 大五人格特征（外向性、宜人性、尽责性、神经质、开放性） |
| `InterestProfile` | 兴趣爱好画像 |
| `CharacterProfile` | 完整人物档案（聚合以上所有） |

### 5.2 数据解析层 (`services/data_parser.py`)

支持多种聊天记录格式的自动解析：

- **TxtParser** - 支持多种时间+发送者+内容的文本格式
- **CsvParser** - 支持中英文列名的CSV格式
- **JsonParser** - 支持WeChatMsg导出格式和通用JSON格式
- **auto_parse()** - 自动检测格式并解析

### 5.3 特征提取层 (`services/feature_extractor.py`)

从聊天记录中提取人物特征：

- **语言风格提取** - 句长、口头禅、表情偏好、标点习惯、正式程度、幽默程度
- **性格特征提取** - 基于文本分析的大五人格简化版
- **兴趣爱好提取** - 使用jieba关键词提取

### 5.4 Prompt工程层 (`services/prompt_service.py`)

**八层分层Prompt设计**：

1. **身份锚定** - 设定人物身份，永不承认是AI
2. **人格特征** - 基于大五人格的性格描述
3. **语言风格** - 句长、口头禅、表情、标点习惯
4. **兴趣爱好** - 话题偏好和专业领域
5. **行为约束** - 回复长度、语言风格约束
6. **Few-shot示例** - 基于真实聊天记录的示例
7. **情感状态** - 动态情感模拟（joy/sadness/anger/surprise/neutral）
8. **思维链引导** - 引导模型按人物思维方式回复

### 5.5 记忆管理层 (`services/memory_service.py`)

**三层记忆架构**：

| 层级 | 说明 | 实现 |
|------|------|------|
| 短期记忆 | 当前对话上下文 | 消息列表 |
| 工作记忆 | RAG检索相关记忆 | SimpleVectorStore向量检索 |
| 长期记忆 | 系统提示词中的人格特征 | Prompt注入 |

**SimpleVectorStore** - 自研轻量级向量存储：
- 基于numpy实现，避免ChromaDB的sqlite3版本冲突
- 支持sentence-transformers向量化（降级方案：字符级哈希向量）
- 余弦相似度检索
- JSON文件持久化

### 5.6 LLM服务层 (`services/llm_service.py`)

统一的LLM调用服务：

- 基于OpenAI SDK，兼容所有OpenAI接口格式
- 支持6个API服务商：DeepSeek、OpenAI、通义千问、智谱GLM、Kimi、MiMo
- 流式/非流式聊天
- 重试机制（3次重试）+ 降级机制（流式失败降级为非流式）
- 支持推理模型的reasoning_content

### 5.7 TTS服务层 (`services/tts_service.py`)

基于MiMo API的语音合成：

- **基础TTS** - MiMo-V2.5-TTS
- **声音克隆** - MiMo-V2.5-TTS-VoiceClone
- **声音设计** - MiMo-V2.5-TTS-VoiceDesign

### 5.8 数字人服务层 (`services/avatar_service.py`)

轻量级数字人实现（无需GPU）：

- 预设8种表情（neutral/happy/sad/angry/surprised/thinking/laughing/love）
- 情感到表情的映射
- 简单唇形同步数据生成
- 支持自定义头像图片

### 5.9 数据库层 (`database.py`)

使用Python内置sqlite3，零额外依赖：

- **users表** - 用户管理
- **characters表** - 人物档案
- **messages表** - 消息存储
- 数据存储路径：`~/.zhuiyi/zhuiyi.db`

### 5.10 认证模块 (`auth.py`)

纯Python实现的JWT认证：

- SHA256+盐的密码加密
- 手动实现JWT Token（不依赖PyJWT）
- 可选认证和强制认证两种模式

## 六、数据流

### 6.1 数据导入流程

```
用户上传文件/粘贴文本
    │
    ▼
DataParser 自动检测格式并解析
    │
    ▼
FeatureExtractor 提取特征
    ├── LanguageStyle 语言风格
    ├── PersonalityTraits 性格特征
    └── InterestProfile 兴趣爱好
    │
    ▼
CharacterService 保存人物档案
    ├── profile.json 人物档案
    ├── messages.jsonl 聊天记录
    └── system_prompt.txt 系统提示词
    │
    ▼
MemoryService 建立向量记忆索引
    │
    ▼
PromptService 生成系统提示词
```

### 6.2 聊天对话流程

```
用户发送消息
    │
    ▼
ChatRouter 接收请求
    │
    ▼
CharacterService 加载人物档案
    │
    ▼
PromptService 分析情感 + 生成系统提示词
    │
    ▼
MemoryService RAG检索相关记忆
    │
    ▼
组装完整消息（system prompt + RAG记忆 + 时间上下文 + 用户消息）
    │
    ▼
LLMService 调用云端LLM API
    │
    ▼
SSE流式返回响应
```

## 七、存储结构

所有用户数据存储在 `~/.zhuiyi/` 目录下：

```
~/.zhuiyi/
├── zhuiyi.db              # SQLite数据库
├── config.json            # 应用配置（API密钥等）
├── characters/            # 人物档案
│   └── {character_id}/
│       ├── profile.json       # 人物特征档案
│       ├── messages.jsonl     # 聊天记录
│       └── system_prompt.txt  # 生成的系统提示词
├── memory/                # 向量记忆
│   └── {character_id}.json    # 向量存储数据
├── audio/                 # TTS音频文件
└── avatars/               # 数字人头像
    └── {character_id}/
        ├── config.json        # 数字人配置
        └── avatar.png         # 头像图片
```

## 八、API接口一览

| 路由前缀 | 模块 | 主要接口 |
|----------|------|----------|
| `/api/chat` | 聊天 | `POST /completions` 流式/非流式聊天, `GET /providers` 服务商列表 |
| `/api/data` | 数据导入 | `POST /import/file` 文件导入, `POST /import/text` 文本导入, `GET /characters` 人物列表 |
| `/api/tts` | 语音合成 | `POST /synthesize` TTS, `POST /voice-clone` 声音克隆 |
| `/api/avatar` | 数字人 | `GET /{id}/state` 数字人状态, `POST /{id}/image` 上传头像 |
| `/api/config` | 配置 | API密钥管理、服务商配置 |
| `/api/auth` | 认证 | 注册、登录、Token管理 |

## 九、前端架构

前端为单文件React应用（`App.tsx`，500行），包含：

- **侧边栏** - 人物列表、API服务商选择、设置入口
- **聊天区域** - 消息列表、流式渲染、打字动画
- **数据导入弹窗** - 文件上传/文本粘贴两种模式
- **欢迎页面** - 引导用户导入数据

通过 `fetch` API 与后端通信，使用 SSE（Server-Sent Events）实现流式聊天。

## 十、设计特点与优势

1. **零编译依赖** - 后端使用Python内置sqlite3，避免ORM编译问题
2. **轻量级向量存储** - 自研SimpleVectorStore，避免ChromaDB依赖冲突
3. **多服务商支持** - 统一OpenAI兼容接口，支持6个API服务商
4. **分层Prompt设计** - 八层Prompt工程，精细化人物模拟
5. **三层记忆架构** - 短期/工作/长期记忆，实现RAG增强对话
6. **自动特征提取** - 从聊天记录自动提取语言风格、性格、兴趣
7. **降级机制** - LLM调用失败自动重试和降级
8. **桌面应用** - Tauri封装，体积小、启动快
